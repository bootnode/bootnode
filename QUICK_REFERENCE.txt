================================================================================
                            NAVIGATION SUMMARY
================================================================================

When user visits: https://web3.hanzo.ai/login

What happens:
  1. Browser loads page (HTTP 200 OK)
  2. Page automatically redirects to: https://hanzo.id/login
  3. OAuth2 parameters included: redirect_uri & state
  4. Hanzo login form displayed
  5. User enters credentials and signs in
  6. Redirect back to web3.hanzo.ai/auth/callback (with auth code)
  7. Backend exchanges code for token
  8. Redirect to https://web3.hanzo.ai/dashboard

Time to login page: < 1 second
UX Experience: Seamless, no intermediate loading screens

================================================================================
                            OAUTH2 PARAMETERS
================================================================================

From URL: https://hanzo.id/login?redirect_uri=...&state=...

Parameter Breakdown:
  redirect_uri: https://web3.hanzo.ai/auth/callback
    └─ Where user returns after authentication
  
  state: /dashboard
    └─ CSRF protection + post-login destination
  
  (client_id, response_type, scope are configured server-side)

Flow Type: OAuth2 Authorization Code
  └─ Most secure flow for web applications
